# CN-JACKFRUIT-MINI-PROJECT

**Project Overview**
This project implements a Telemetry Collection System where multiple distributed clients continuously send sensor data to a centralized server using UDP socket programming.
The system simulates real-world telemetry applications such as IoT monitoring, where data like temperature, humidity, and battery level is transmitted and analyzed in real time.

**Objectives**
Design a multi-client telemetry system
Implement UDP-based communication
Perform performance evaluation (throughput)
Demonstrate scalability and failure handling
Apply basic security mechanisms


**Technologies Used**
Python
Socket Programming
UDP Protocol
Base64 Encoding


**Features Implemented**
Phase 1 (Core Implementation)
UDP client-server communication
Real-time telemetry data transmission
Multiple client support
Structured data format (Device ID, Timestamp, Sensor values)


**Phase 2 (Advanced Features)**
Performance Evaluation
Throughput calculation (packets/sec)
Scalability Testing
Multiple clients sending data simultaneously
Failure Handling
Handles corrupted/invalid data using exception handling
Optimization
Efficient data transmission using UDP
Controlled sending rate

**Performance Metrics**
**Throughput Calculation**
Throughput = Number of Packets / Time (seconds)

**Network Requirements**
Both systems must be on the same network
Recommended: Mobile Hotspot

**Key Concepts Demonstrated**
UDP Socket Programming
Real-time Data Transmission
Distributed Systems
Performance Analysis
Basic Network Security

**Conclusion**
This project successfully demonstrates a scalable telemetry system using UDP, capable of handling multiple clients, evaluating performance, and ensuring reliable communication with basic security mechanisms.
